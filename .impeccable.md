# LibreFunnels Design Context

## Design Context

### Users
LibreFunnels is designed first for WooCommerce store owners who need to build high-converting funnels without paid funnel software. Secondary users include agencies, creators, and developers, but the first-run admin experience must be clear enough for a non-technical store owner managing real products, shipping, taxes, gateways, and offers.

### Brand Personality
LibreFunnels should feel refined, trustworthy, and capable. The product is open-source and free, but it should not feel cheap, improvised, or like a generic WordPress settings plugin.

### Aesthetic Direction
Use a refined SaaS direction: premium, calm, conversion-focused, and easy to scan. The admin UI should feel like a polished funnel-building workspace inside WordPress, not a cluttered meta-box maze. Visual polish should clarify structure: starting step, route labels, missing configuration, and next actions.

### Design Principles
- Clarity before decoration: every visual choice should help users understand funnel structure or status.
- Store-owner language: labels and empty states should explain outcomes, not implementation details.
- Canvas first, confusion never: routes, accept/reject paths, and broken states must be visually explicit.
- Progressive depth: common actions should be obvious; advanced controls should be available without crowding the first screen.
- WordPress-compatible, not WordPress-generic: respect admin conventions while giving LibreFunnels a distinctive product feel.
